# 小市值 + 波动率双因子回测设计

## 1. 目标

在 `backtest/small_market_capital_2.py` 的现有小市值选股框架上，引入波动率因子，形成双因子打分选股：

- 市值因子权重：60%
- 波动率因子权重：40%
- 波动率窗口：20 日
- 每月首个交易日调仓、等权买入 20 只股票（保持原策略节奏）

## 2. 当前策略基线

现有策略流程：

1. 月度调仓，3/4/12 月空仓。
2. 候选池来自 A 股（排除北交所等），并过滤 ST、PB/PE<=0、上市时间不足约 2 年。
3. 按总市值升序取前 `top_n`，再等权买入前 `buy_count`。

本次改造保持过滤条件、调仓频率和交易节奏不变，仅替换“最终选股规则”。

## 3. 双因子打分方案

### 3.1 因子定义

- 市值因子（越小越好）：
  - 基于 `COL_TOTAL_MV` 做截面分位排名并转分数。
- 波动率因子（越低越好）：
  - 对每只股票取最近 20 个交易日收盘价计算日收益率；
  - 波动率 = `pct_change(close).rolling(20).std()`；
  - 在当期截面中做分位排名并转分数。

### 3.2 综合打分

`total_score = 0.6 * mv_score + 0.4 * vol_score`

按 `total_score` 降序选出前 `buy_count` 只作为当期持仓。

## 4. 数据流设计

1. 进入 `rebalance(current_date)` 后，先执行现有 SQL 得到当日候选基础表。  
2. 针对候选股票批量读取到 `current_date` 的近 20 日历史收盘价。  
3. 逐股票计算波动率，并 merge 回候选基础表。  
4. 计算 `mv_score`、`vol_score`、`total_score`。  
5. 选前 20 并执行等权调仓。

## 5. 异常与边界处理

- 股票历史窗口不足 20 日：该股票波动率记为缺失，并在当次评分中剔除。  
- 收盘价异常（无法转数值）：同样剔除。  
- 当期有效股票数量 < `buy_count`：按有效数量等权买入。  
- 当期有效股票过少（如接近 0）：打印告警，跳过本次调仓并保持当前仓位。

## 6. 参数建议

在策略参数中新增：

- `vol_window = 20`
- `weight_mv = 0.6`
- `weight_vol = 0.4`
- `score_method = "rank_pct"`

保留原参数：

- `buy_count = 20`
- `min_list_days = 730`

## 7. 测试与验收

1. **打分单元测试**：给定模拟截面，校验分位分数与综合分数计算正确。  
2. **策略行为测试**：校验最终持仓数量、缺失波动率股票被剔除。  
3. **回归对比测试**：同一回测区间对比改造前后持仓名单，验证双因子逻辑已生效。  

验收标准：

- 策略可稳定运行，不破坏原调仓节奏；  
- 综合打分权重按 60/40 生效；  
- 组合持仓与原纯小市值策略相比出现可解释变化。
