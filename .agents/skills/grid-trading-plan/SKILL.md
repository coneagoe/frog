# Grid Trading Plan Skill

把用户的网格交易请求转成可执行、风险受控、不过度武断的交易计划。核心原则：网格赚震荡的钱，不是单边下跌时无限补仓的理由。

## 使用边界

- 必须说明：内容仅供研究和交易计划整理，不构成个性化投资建议，不保证收益。
- 不自动下单，不承诺胜率，不暗示“必涨/必赚”。
- 不伪造实时行情；若未取到实时数据，明确“以用户交易软件实时价格为准”。
- 每次生成具体网格价位、买卖档位、预警线或硬止损线前，必须重新获取或重新计算最新可用数据；不能沿用旧回答、静态示例或记忆中的价格。
- 若数据源不可用，必须先说明“未重新取数/未重新计算”，只能输出不含固定价位的通用公式或模板，除非用户明确要求使用其提供的参考价。
- 若用户没有给成本、资金、风险偏好，可先给通用模板；但必须列明假设和缺失信息，固定价格止损必须标明是基于当前参考价的示例。

## 重新计算要求

任何包含固定价格的网格计划都必须先完成重新计算，并在输出中写明数据口径：

1. 标的代码与市场。
2. 数据源（例如 Tushare、本地数据库、用户提供价格）。
3. 最新交易日或参考时间。
4. 当前价或最新收盘价。
5. 计算依据：近期区间、波动率/ATR、或用户指定区间。

不能把示例价位当成当次计划。即使同一标的刚刚回答过，只要用户要求“计划/档位/止损线”，也要重新取最新可用数据或至少重新计算并声明口径。

## 信息收集

优先确认或自行说明缺失：

| 信息 | 用途 |
| --- | --- |
| 标的代码与市场 | 避免简称歧义 |
| 当前价或参考价 | 计算网格区间和止损价 |
| 已有仓位与成本 | 区分已有持仓和空仓新开 |
| 计划投入资金 | 计算每格资金和仓位上限 |
| 最大可承受亏损 | 反推硬止损线 |
| 周期 | 区分短线网格、波段网格、长期底仓 |

## 默认输出结构

1. 一句话结论：是否适合做网格，以及核心风险。
2. 数据口径：说明价格、时间范围、是否延迟。
3. 网格区间：给上沿、下沿、暂停补仓条件。
4. 仓位分层：底仓比例、网格资金份数、单票仓位上限。
5. 买卖档位：列出买入档和卖出档，或给公式。
6. 止损规则：预警线、硬止损线、趋势止损条件。
7. 执行纪律：不追高、不加倍、不突破规则手动补仓。
8. 风险提示：题材、业绩、流动性、单边下跌风险。

## 多标的网格适配度筛选

当用户问“哪些股票适合做网格 / 多只股票里怎么选 / 网格候选筛选”时，优先使用仓库脚本固化计算过程，不要每次在对话里临时生成 Python：

```bash
uv run python tools/grid_trading_screener.py "药明康德=603259.SH" "中国神华=601088.SH" "长江电力=600900.SH"
```

默认取最近约 `460` 个自然日并使用最新 `250` 条日线；如需固定口径，可传 `--start-date YYYYMMDD --end-date YYYYMMDD`。

也可以从 CSV 读取，CSV 至少包含 `name,code` 两列：

```bash
uv run python tools/grid_trading_screener.py --input-csv candidates.csv --output-csv grid_screen.csv
```

脚本会固化这些指标：

| 指标 | 用途 |
| --- | --- |
| `range_pct` | 近一年价格区间振幅，判断有没有网格空间 |
| `annual_vol` | 年化波动率，判断网格间距应宽还是窄 |
| `trend_eff` | 趋势效率，越低越偏震荡，越适合网格 |
| `max_drawdown` | 最大回撤，识别越跌越补的风险 |
| `pos` | 当前价处于近一年区间的位置，避免高位开网格 |
| `avg_amount_yi` | 近段成交额，过滤流动性不足标的 |
| `score/category` | 综合分组：`更适合`、`可做但谨慎`、`不优先` |

筛选输出规则：

1. 必须写明数据源、最新交易日、样本窗口和是否成功取数。
2. 只做筛选时，不生成固定买入/卖出档位；若用户继续要某只票的档位，回到单票网格计划流程重新计算。
3. 输出按 `更适合`、`可做但谨慎`、`不优先` 分组，而不是只按分数机械排名。
4. 高波动题材股即使评分高，也要标注“小仓位、宽网格、硬止损”。
5. 低波红利股要提示“网格空间小，可能更像配置仓”，不要强行给密集网格。
6. 趋势/产业贝塔很强的股票要提示“可能卖飞或越跌越补”，不把高波动等同于适合。

## 参数参考

| 场景 | 网格间距 | 初始仓位 | 网格份数 | 单票上限 |
| --- | --- | --- | --- | --- |
| 波动中等 | 3%–4% | 30%–40% | 6–8 份 | 10%–20% 总账户 |
| 波动较大 | 4%–6% | 20%–30% | 8–10 份 | 10%–15% 总账户 |
| 已有高仓位 | 只减不加或小额加 | 先降风险 | 4–6 份 | 按最大亏损倒推 |

## 止损设计

- 预警线：跌破网格下沿或关键支撑后，停止新增买入。
- 硬止损线：通常设在网格下沿再下跌 5%–8%，或用账户最大亏损倒推。
- 账户风险：单票最大亏损一般不超过总账户 2%–3%。
- 趋势止损：有效跌破中长期均线且反抽无力时，网格失效。
- 若跌破硬止损后三日内不能重新站回预警线，不急于接回。

## 巨人网络示例

当用户问“巨人网络网格交易计划”且没有提供更多信息时，必须先尝试重新获取或重新计算 `002558.SZ` 的最新可用数据。只有在数据源不可用时，才可给不含固定价位的通用模板，并提醒价格需按实时行情更新。

禁止直接沿用以下历史示例价位作为当前计划；这些价位只能用于说明格式，不能替代当次重新计算：

- 参考标的：巨人网络 `002558.SZ`。
- 若当前价约在 `27` 元附近，可考虑中等偏宽网格，参考区间 `24.20`–`31.80` 元。
- 初始仓位：计划资金的 `30%–40%`；剩余资金分 `6–8` 份。
- 买入档示例：`26.20`、`25.20`、`24.20`、`23.30`。
- 卖出档示例：`28.30`、`29.40`、`30.60`、`31.80`。
- 预警线：日收盘跌破 `24.20`，停止新增买入。
- 硬止损线：日收盘跌破 `22.80`，网格失效，考虑退出或减至观察仓，具体执行需用户自行确认。

## 常见错误

- 没有重新取数/重新计算，就直接复用旧回答或 skill 示例里的固定网格价位。
- 直接给固定止损价但不说明参考价格和数据延迟。
- 忽略用户成本、资金规模和最大亏损承受能力。
- 把网格当成越跌越买，而不是有失效条件的震荡策略。
- 网格过密，收益被交易成本和噪音吃掉。
- 突破上沿后仍机械卖光，错过趋势行情；跌破下沿后仍机械补仓，放大亏损。

## 输出风格

- 中文回答优先使用清晰小标题和短列表。
- 给价格时写“参考/示例/以实时行情为准”。
- 用户只要计划时，直接给计划；若关键信息缺失，不要卡死流程，可先给保守模板并列出需要补充的信息。
